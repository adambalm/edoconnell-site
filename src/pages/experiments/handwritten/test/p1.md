# Handwriting Pipeline — Loop Test

This page exists to prove that a markdown file in
`src/pages/experiments/handwritten/<slug>/<page>.md` reaches a public URL on
edoconnell.org via the existing Astro + Vercel deploy. No Sanity, no auth, no
build-time sync script — just file → commit → push → URL.

If you can read this, the loop closed.

## What it replaces

A real handwritten Supernote page transcribed by `ghost-writer-v3/pipeline/supernote_pipeline.py` would land at the same path with frontmatter like:

```yaml
title: "<LLM-generated page title>"
date: <transcription date>
source: <notebook>.note
page: <page number>
publish: true
```

The `publish: true` flag is the agreed gating mechanism (see
`~/.claude/plans/cold-start-2026-04-22-handwriting-verbatim.md`, Turn 3, lines
136–154). For now the flag is decorative — the file's presence under
`src/pages/experiments/` is what publishes it. A future build-time sync
script will read flagged pages from BasicMemory and copy them in.

## What's deliberately not here yet

- Per-notebook index page (`/experiments/handwritten/<slug>/`)
- `/experiments/handwritten/` index of all notebooks
- Pagefind search
- Side-by-side flat-PNG rendering of the original page
- Authentication / gating (the cold-start design says publication itself is
  the gate)

These are the four discrete additions the cold-start doc proposed; this test
covers the route plumbing only.
